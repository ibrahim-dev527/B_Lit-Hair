⚠️ Required: Grant Table Access Run once if site shows no data

Tables created via the SQL Editor do NOT automatically give the website permission to read them. If your shop or services page shows no data, run this SQL block once in your Supabase SQL Editor. You only need to do this once.

-- Run this in Supabase SQL Editor (once only) -- This grants the website permission to read your tables grant usage on schema public to anon, authenticated; grant select on products to anon; grant select on services to anon; grant select on reviews to anon; grant insert on contact_submissions to anon; -- For admin full access (also run this): grant all on products to authenticated; grant all on services to authenticated; grant all on reviews to authenticated; grant all on contact_submissions to authenticated;

📸 Image Storage Setup One-time setup

To upload product photos from the admin panel, create a public storage bucket. Run this SQL once in your Supabase SQL Editor. Until you do this, add product images using an image URL from Cloudinary or ImgBB instead.

-- Run this in Supabase SQL Editor insert into storage.buckets (id, name, public) values ('product-images', 'product-images', true); create policy "Anyone can upload product images" on storage.objects for insert with check (bucket_id = 'product-images'); create policy "Public can read product images" on storage.objects for select using (bucket_id = 'product-images');
