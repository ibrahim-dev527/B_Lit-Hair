Create the Database Tables

These tables store your products, services, reviews and messages

1
Open SQL EditorIn Supabase click SQL Editor in the left sidebar.
2
Click "New query"This opens a blank text area where you paste code.
3
Copy, paste and run each block belowDo one block at a time. Paste it, click the green Run button, wait for success, then do the next block.
Table 1 of 4 — Products
create table products (
  id text primary key,
  name text not null,
  category text not null check (category in ('wigs','bundles','frontals','extensions')),
  price numeric not null,
  currency text default 'GHS',
  badge text,
  hair_type text,
  texture text,
  default_length text,
  weight text,
  color text,
  description text,
  features jsonb default '[]',
  image text,
  images jsonb default '[]',
  whatsapp_msg text,
  is_visible boolean default true,
  sort_order integer default 0,
  created_at timestamptz default now()
);
Table 2 of 4 — Services
create table services (
  id text primary key,
  name text not null,
  description text,
  starting_price numeric,
  currency text default 'GHS',
  duration text,
  icon text,
  tags jsonb default '[]',
  is_visible boolean default true,
  sort_order integer default 0,
  created_at timestamptz default now()
);
Table 3 of 4 — Reviews
create table reviews (
  id bigint generated always as identity primary key,
  reviewer_name text not null,
  reviewer_location text,
  stars integer default 5 check (stars between 1 and 5),
  review_text text not null,
  is_approved boolean default false,
  created_at timestamptz default now()
);
Table 4 of 4 — Contact Submissions
create table contact_submissions (
  id bigint generated always as identity primary key,
  name text not null,
  email text not null,
  phone text,
  subject text,
  message text not null,
  is_read boolean default false,
  created_at timestamptz default now()
);

Set Row Level Security (RLS)

This controls who can read and write to your database

In SQL Editor click New query, paste the entire block below and click Run. Run this as one block.

Row Level Security — run the entire block at once
-- Products
alter table products enable row level security;
create policy "Public read products"
  on products for select using (is_visible = true);
create policy "Admin full access products"
  on products for all using (true) with check (true);

-- Services
alter table services enable row level security;
create policy "Public read services"
  on services for select using (is_visible = true);
create policy "Admin full access services"
  on services for all using (true) with check (true);

-- Reviews
alter table reviews enable row level security;
create policy "Public read approved reviews"
  on reviews for select using (is_approved = true);
create policy "Admin full access reviews"
  on reviews for all using (true) with check (true);

-- Contact Submissions
alter table contact_submissions enable row level security;
create policy "Anyone can submit contact"
  on contact_submissions for insert with check (true);
create policy "Admin full access contacts"
  on contact_submissions for all using (true) with check (true);

Grant Table Access to the Website

This is the step most people miss — without it, the shop and services pages will stay empty

🚨
Why This Step is Critical

When you create tables using the SQL Editor, Supabase does NOT automatically give your website permission to read them. RLS policies alone are not enough — you must also GRANT the anon role access to the table itself. Without this step, your shop and services pages will always show empty, even if you've added products and services in the admin.

In SQL Editor click New query, paste the block below and click Run.

Grant Access — run this entire block at once
-- Allow the website (anon role) to read your tables
grant usage on schema public to anon, authenticated;

grant select on products to anon;
grant select on services to anon;
grant select on reviews to anon;
grant insert on contact_submissions to anon;

-- Allow the admin panel full access
grant all on products to authenticated;
grant all on services to authenticated;
grant all on reviews to authenticated;
grant all on contact_submissions to authenticated;

-- Allow sequences (needed for auto-increment IDs)
grant usage, select on all sequences in schema public to anon, authenticated;
✅
What this does

This gives the website permission to read your products, services and approved reviews. It also allows anyone to submit the contact form. The admin panel gets full control over everything. Without this, all pages would return empty results even with the correct API keys.

📦 Products Tips

  • For the image: upload your photo to imgbb.com (free) and paste the link into the Image URL field
  • Products with a Badge (e.g. "Best Seller") appear in the featured section on the homepage
  • Sort Order controls display order — lower number = shows first
  • You can have as many products as you want for free

✂️ Services Tips

  • Add as many services as you offer — hair sales, braiding, styling, bridal, wigs etc.
  • The icon field uses a simple name like star, scissors, wig — pick from the dropdown in the form
  • Use the Tags field to show styles you offer (e.g. "Box Braids", "Knotless")
  • Set Sort Order to control which service appears first

⭐ Reviews Tips

  • New reviews are hidden by default — you must click Approve to show them
  • Only approved reviews appear on the website
  • Add reviews directly from good WhatsApp feedback you receive
  • Aim for at least 6 approved reviews to look credible

📧 Messages Tips

  • Unread messages show a count badge in the sidebar
  • Messages are marked as read automatically when you view them
  • If the customer gave a phone number, you can reply directly on WhatsApp
  • Click Refresh to check for new messages
