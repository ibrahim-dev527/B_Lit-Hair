Create the Database Tables

These tables are where your products, services, reviews and messages are stored

1
Open the SQL Editor in Supabase In your Supabase project, click SQL Editor in the left sidebar. It has a code icon that looks like </>.
2
Click "New query" This opens a blank text area where you will paste the code below.
3
Copy each block below, paste it, and click the green RUN button Do this separately for each of the 4 tables. Copy one block → paste → click Run → check it says "Success" → then do the next one.
Table 1 of 4 — Products (stores your hair products)
create table products (
  id text primary key,
  name text not null,
  category text not null check (category in ('wigs','bundles','frontals','extensions')),
  price numeric not null,
  currency text default 'GHS',
  badge text,
  hair_type text,
  texture text,
  default_length text,
  weight text,
  color text,
  description text,
  features jsonb default '[]',
  image text,
  images jsonb default '[]',
  whatsapp_msg text,
  is_visible boolean default true,
  sort_order integer default 0,
  created_at timestamptz default now()
);
Table 2 of 4 — Services (stores your hair services)
create table services (
  id text primary key,
  name text not null,
  description text,
  starting_price numeric,
  currency text default 'GHS',
  duration text,
  icon text,
  tags jsonb default '[]',
  is_visible boolean default true,
  sort_order integer default 0,
  created_at timestamptz default now()
);
Table 3 of 4 — Reviews (stores customer reviews)
create table reviews (
  id bigint generated always as identity primary key,
  reviewer_name text not null,
  reviewer_location text,
  stars integer default 5 check (stars between 1 and 5),
  review_text text not null,
  is_approved boolean default false,
  created_at timestamptz default now()
);
Table 4 of 4 — Contact Submissions (stores messages from your contact page)
create table contact_submissions (
  id bigint generated always as identity primary key,
  name text not null,
  email text not null,
  phone text,
  subject text,
  message text not null,
  is_read boolean default false,
  created_at timestamptz default now()
);
✅
How to know it worked

After clicking Run for each block, you should see a green message that says something like "Success. No rows returned." If you see an error message in red, read it carefully — it usually tells you exactly what went wrong.

6

Set Access Permissions

This controls who can read and write to your database — run this once

Still in the SQL Editor — click New query again, then copy the entire block below and click Run. Run this all at once as a single block.

-- Products: public can read visible ones, admin can do everything
alter table products enable row level security;
create policy "Public read visible products"
  on products for select using (is_visible = true);
create policy "Admin manage products"
  on products for all using (true) with check (true);

-- Services: public can read visible ones, admin can do everything
alter table services enable row level security;
create policy "Public read visible services"
  on services for select using (is_visible = true);
create policy "Admin manage services"
  on services for all using (true) with check (true);

-- Reviews: public can read approved ones, admin can do everything
alter table reviews enable row level security;
create policy "Public read approved reviews"
  on reviews for select using (is_approved = true);
create policy "Admin manage reviews"
  on reviews for all using (true) with check (true);

-- Contact submissions: anyone can send, admin can read and manage
alter table contact_submissions enable row level security;
create policy "Anyone can submit contact form"
  on contact_submissions for insert with check (true);
create policy "Admin manage contact submissions"
  on contact_submissions for all using (true) with check (true);
💡
What this does

This block sets rules so that: visitors can only see visible products and approved reviews. Only your admin dashboard can add, edit or delete anything. Contact form messages are saved securely and only you can read them.

7

Set Up Image Storage

This allows product photos you upload in the admin to be stored permanently in the cloud

📸
Why this matters

When you upload a product photo in the admin, the system tries to save it to Supabase Storage (cloud). If this step is not done, photos are saved as compressed image data directly in the database — this still works but is slower for larger photos. Setting up Storage gives you better performance and unlimited image size.

In your Supabase SQL Editor, click New query, paste this block, and click Run:

-- Create a public storage bucket for product images
insert into storage.buckets (id, name, public)
values ('product-images', 'product-images', true);

-- Allow anyone to upload images
create policy "Public can upload product images"
  on storage.objects for insert
  with check (bucket_id = 'product-images');

-- Allow anyone to view images
create policy "Public can view product images"
  on storage.objects for select
  using (bucket_id = 'product-images');
✅
After running this

When you upload a photo in the admin dashboard, it will be saved to Supabase Storage and get a permanent cloud URL. This URL is what gets stored in your database and shown on the website — so your product photos will always load fast.

💡
Can also be done from Settings in your admin

Open admin.html → go to Settings → you'll see this same SQL code with a "Copy" button ready to use.
